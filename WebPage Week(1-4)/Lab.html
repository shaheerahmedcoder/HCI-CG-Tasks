Exam Practice Table

The following table identifies common errors in semantic HTML and accessibility, specifically focused on the requirements for Lab 01.

Common HTML structure and accessibility mistakes with corrections
UI Element / Issue Wrong HTML (with mistake) Correct UI Element Correct HTML
Duplicate h1 tags 
<header><h1>Project Overview</h1></header>
<section><h1>Task List</h1></section>
 Use one h1 per page; h2 or h3 for subsections. 
<h1>Project Overview</h1>
<h2>Task List</h2>
Images without alt 
<img src="chart.png">
 All images should have descriptive alt text. 
<img src="chart.png" 
alt="Bar chart showing weekly user interactions">
Incorrect heading order 
<h3>Step 1</h3>
<h2>Introduction</h2>
 Headings must follow a logical sequence. 
<h2>Introduction</h2>
<h3>Step 1</h3>
Using div instead of semantic tags 
<div class="footer">...</div>
<div class="menu">...</div>
 Use <footer> and <nav> for proper semantics. 
<footer>...</footer>
<nav>...</nav>
Vague links 
<a href="interaction.html">Learn more</a>
 Use descriptive link text for clarity and accessibility. 
<a href="interaction.html">Detailed Interaction Analysis</a>
Table headers missing scope 
<tr>
  <th>Task</th>
  <th>Time</th>
</tr>
 Add scope="col" or scope="row" to header cells. 
<tr>
  <th scope="col">Task</th>
  <th scope="col">Time</th>
</tr>
Label not linked to input 
<label>Email:</label>
<input type="email" id="email">
 Labels should use the for attribute to associate with the input ID. 
<label for="email">Email:</label>
<input type="email" id="email">
Navigation links not in a list 
<nav>
  <a href="home.html">Home</a>
  <a href="about.html">About</a>
</nav>
 Navigation should be structured using ul > li. 
<nav>
  <ul>
    <li><a href="home.html">Home</a></li>
    <li><a href="about.html">About</a></li>
  </ul>
</nav>

Course Progress Journal

This journal summarizes key concepts from Weeks 1–4 of HCI based on *Human–Computer Interaction* (3rd Edition) by Dix et al.

Week 1: Human Memory and Foundations

Week 1 focused on how humans perceive and store information, including memory types and the implications for interaction design. In HCI, understanding human memory is essential because memory limitations directly shape how interfaces should be organized.

  • Human Memory: Memory is categorized into sensory, short-term, and long-term stores, each impacting how users retain information while interacting with systems.
  • Retention Strategies: Meaningful learning and rehearsal help move information into long-term memory, improving usability.
  • Cognitive Constraints: Designers must account for limited working memory to avoid overloading users.

Miller’s 7 ± 2 Rule: Short-term memory can hold roughly 7 ± 2 chunks of information, which guides minimalist interface design.

Week 2: UI Design Principles & Cognitive Models

In Week 2, we explored various design principles and models that help structure user interfaces logically and intuitively.

  • Design Fundamentals: Effective interfaces balance consistency, ease of use, and relevant feedback to users.
  • Semantic Networks: Concepts in long-term memory are linked to other concepts, aiding efficient retrieval during interaction.
  • Forgetting and Retrieval: Designers need to reduce unnecessary cognitive load because forgetting can occur through interference and decay.

Norman’s Interaction Steps

Norman’s model outlines seven stages of interaction that users go through—from setting a goal to evaluating feedback—providing a framework to analyze user behavior.

Week 3: Quiz and Applied Scenarios

Week 3 involved a quiz that reinforced earlier models and encouraged applying them to scenarios, such as interface tasks seen in common applications.

Key Highlights:

  • Review of Interaction Models: Applying Norman’s interaction cycle to simple case studies.
  • Concept Integration: Combining principles from cognitive psychology and design models to evaluate interfaces.

Week 4: Shneiderman’s Rules and GOMS

In the final week of this assignment, we studied structured guidelines for interface design, including Shneiderman’s principles and the GOMS model for task analysis.

Shneiderman’s Eight Golden Rules:

  1. Consistency across the interface.
  2. Offer shortcuts for experienced users.
  3. Display informative feedback.
  4. Design dialogs to give closure.
  5. Help users avoid and recover from errors.
  6. Permit easy reversal of actions.
  7. Promote internal locus of control.
  8. Reduce memory load wherever possible.

GOMS Overview:

GOMS (Goals, Operators, Methods, Selection rules) represents a cognitive model for predicting how users achieve tasks and the time required, guiding task-level interface analysis.

References: Alan Dix, Janet Finlay, Gregory D. Abowd & Russell Beale, Human–Computer Interaction, 3rd Edition, Pearson Education, 2004.

Norman and GOMS: Interaction-Level Comparison

Scenario: A user wants to record a shared dinner expense of $50 that they paid on behalf of a friend named “Shaheer” using a mobile expense tracking application.

Mapping Norman’s Action Cycle to GOMS elements for a shared-expense task
Norman’s Stage Observed User Behavior GOMS Perspective Analytical Relationship
Goal ↔ GOAL The user decides to log the payment so the expense is properly tracked and reimbursed later. GOAL: RECORD-EXPENSE.
Represents the overall intention motivating the task. 		✅ Strong Alignment
Both models begin with a clearly defined objective.
Intention / Plan ↔ METHOD The user chooses to use the “Quick Add” option instead of navigating through group menus. METHOD: QUICK-ENTRY.
Selection Rule: If minimal input is required, prefer Quick Add. 		✅ Conceptual Correspondence
GOMS formalizes the decision using rules.
Specify The user visually searches for input fields and decides where to enter the amount and select the friend’s name. Implicit.
GOMS assumes the user already knows the interface layout. 		Key Difference: Norman captures exploratory cognition; GOMS abstracts it away.
Execute ↔ OPERATORS The user taps buttons, types “50,” selects “Shaheer,” and confirms the action. OPERATORS: K (typing), P (pointing), B (button press). ✅ Direct Mapping
Physical actions correspond precisely to GOMS operators.
Perceive / Interpret / Evaluate 
Perceive: Confirmation message and updated balance appear.
Interpret: The system successfully recorded the expense.
Evaluate: The user feels confident the task is complete.
Not Modeled.
GOMS presumes successful completion once execution ends. 		Critical Distinction: Norman emphasizes feedback and closure of the Gulf of Evaluation.

Analytical Insight

For the “Who Paid” expense task, Norman’s model is particularly useful for identifying usability problems related to discoverability and feedback. In contrast, GOMS and KLM are more effective when predicting expert performance, such as estimating how quickly multiple expenses can be entered in succession.

This demonstrates why Norman’s model is preferred during early design evaluation, while GOMS/KLM are better suited for optimization and comparison of mature interfaces.

References: Dix, A., Finlay, J., Abowd, G., Beale, R. Human–Computer Interaction, 3rd Edition. Pearson Education, 2004. Chapters 3 (Interaction), 9 (GOMS), and 12 (Predictive Models).
